## Стандартизированный шаблон базы знаний

Это **шаблон**, а не просто документация. Его назначение — служить основой для создания любых баз знаний. Клонируешь, добавляешь свои `.mdx`-файлы в `src/content/docs/` — и всё работает.

### Принципы

**Универсальность.** Функциональность работает на уровне контейнера, а не покомпонентно. Например, click-to-expand реализован через event delegation на `.docs-content` — любой блок (код, диаграмма, таблица, изображение) раскрывается по клику без модификации самих компонентов.

**Предсказуемость.** Новый MDX-файл с корректным frontmatter автоматически появляется в навигации, поиске (Cmd+K) и оглавлении. Никаких дополнительных настроек.

**Темная и светлая темы.** Переключение без визуальных артефактов (белая вспышка устранена). Контраст соответствует WCAG 2.1 AA.

**Статический деплой.** По умолчанию шаблон собирается как статический сайт для GitHub Pages. Серверные функции (редактор, API) отключаются в CI.

### Что поддерживается

| Элемент | Формат в MDX | Статус |
|---------|-------------|--------|
| Заголовки h1-h4 | `# ## ### ####` | Работает |
| Параграфы, списки | стандартный Markdown | Работает |
| Таблицы | GFM таблицы | Работает |
| Code blocks с подсветкой | ` ```language ` | Работает |
| Mermaid-диаграммы | ` ```mermaid ` | Работает |
| Изображения | `![alt](url)` | Работает |
| Callout'ы | JSX-компонент | Работает |
| Внутренние ссылки | `[текст](/docs/slug/)` | Работает |
| Внешние ссылки | `[текст](https://...)` | Работает |

### Как использовать

1. Клонировать репозиторий
2. Установить зависимости: `bun install`
3. Запустить: `bun run dev`
4. Добавить свои `.mdx`-файлы в `src/content/docs/`
5. Деплой: push в GitHub — CI соберёт статический сайт

### Навигация по разделам

Используйте боковую панель слева для перемещения между разделами. Текущее оглавление страницы — справа.
